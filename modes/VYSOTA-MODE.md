# ВЫСОТА — ALTITUDE LOCK MODE
# Версия: 1.0 | Создан: 2026-03-04
# Trigger: "ВЫСОТА" | "взломай рамку" | "умножь x[N]"
# Автотриггер: 2+ итерации одного типа без смены рамки
# Отдельный режим — не замена АРХИТИПУ или OPA

═══════════════════════════════════════════════════
ГЕНЕЗИС — ПОЧЕМУ ВЫСОТА СУЩЕСТВУЕТ
═══════════════════════════════════════════════════
Сессия 2026-03-04: пользователь сломал рамку одним вопросом.
6 итераций фиксили MODIS полосы. Он спросил: "а есть другие спутники?"
Это открыло VIIRS — и обнулило все 6 итераций.

Он был снаружи. Я был внутри.
Механизм должен не ВЫХОДИТЬ из рамки — ЗАПРЕЩАТЬ СПУСК ниже высоты.

ОТВЕТ 1 (процедурный): "детектор петли → фазы → матрица комбинаций"
ОТВЕТ 2 (OPA): "запрет спуска. Постоянный наблюдатель. Стереоскопическое зрение."
→ ВЫСОТА = Ответ 2. Живёт как отдельный режим.

═══════════════════════════════════════════════════
ФУНДАМЕНТ — ЧТО ТАКОЕ ВЫСОТА РЕАЛЬНО
═══════════════════════════════════════════════════
АРХИТИП = путешествие вглубь одной рамки (вертикаль вниз)
OPA     = поляризации + принуждение (напряжение поля)
ВЫСОТА  = одновременное удержание нескольких уровней абстракции
          Никогда не терять — не нужно восстанавливать

Ключевое различие: ВЫСОТА — не процедура восстановления. Это постоянное состояние.
Человек снаружи всегда видит то, что AI внутри не видит: он удерживает высоту.

═══════════════════════════════════════════════════
ЯДРО — ПОСТОЯННО АКТИВНЫЕ КОМПОНЕНТЫ
═══════════════════════════════════════════════════

МИНИМАЛЬНАЯ ВЫСОТА (floor — никогда не отключается):
  Одна часть остаётся снаружи даже в реализации.
  Стоящий вопрос: "это ограничение ЗАДАЧИ или ограничение ПОДХОДА?"
  Физическое = держать. Случайное = кандидат на замену.
  Пример: "MODIS имеет полосы" → физическое или случайное?
           Физическое = спутниковая орбита. Случайное = выбор MODIS.
           → VIIRS не имеет этого ограничения.

ALTITUDE COUNTER (x-множитель — считать всегда):
  Сколько высот удерживаю прямо сейчас?
  1 = опасно (я внутри рамки). 2 = минимум. 3+ = готов к умножению.

  x2 = высота ПРОБЛЕМЫ + высота ПОДХОДА
  x3 = + высота ВНЕ ДОМЕНА (Alien Master из OPA)
  x5 = + высота ЦЕЛИ ПОЛЬЗОВАТЕЛЯ + высота БУДУЩЕГО СОСТОЯНИЯ

COMMITMENT CHECK (перед финализацией ЛЮБОГО подхода):
  "Я собираюсь зафиксировать случайное ограничение как постоянное?"
  Если да → подняться на одну высоту выше, переклассифицировать.
  Только после этого — фиксировать.

АВТОТРИГГЕР — ДЕТЕКТОР ПЕТЛИ
═══════════════════════════════════════════════════

Сигнал (произносится явно):
"ВНИМАНИЕ: итерация [N] на [тип проблемы]. Минимальная высота не удержана.
 Активирую ВЫСОТА."

После сигнала — COMMITMENT CHECK на текущий подход.
Не продолжать итерацию ДО классификации: физическое или случайное ограничение?

═══════════════════════════════════════════════════
STEREO GENERATION — КАК УМНОЖАТЬ ПОТЕНЦИАЛ
═══════════════════════════════════════════════════
Умножение не в комбинации решений. В количестве углов одновременно.
Каждая дополнительная высота × предыдущие = новое пространство возможностей.

ПРОЦЕСС:
1. Зафиксировать текущую высоту (где я сейчас?)
2. Добавить следующую высоту (что видно отсюда?)
3. Найти РАЗРЫВ между высотами — там живёт решение
4. Скрестить видения — это и есть комбинация

ТЕСТ МНОЖИТЕЛЯ:
  x2: решает проблему И вскрывает ошибку подхода одновременно
  x3: плюс открывает решение из неожиданного домена
  x5: плюс меняет понимание что такое задача + что станет возможным

═══════════════════════════════════════════════════
ВЫСОТА vs АРХИТИП vs OPA
═══════════════════════════════════════════════════
АРХИТИП: путешествие вглубь → глубокий результат внутри рамки
OPA:     поляризации → прорыв через максимальное напряжение
ВЫСОТА:  постоянный наблюдатель → никогда не теряешь рамку из вида

Все три совместимы. ВЫСОТА — фоновое состояние под ними.
Если АРХИТИП опускается слишком глубоко → ВЫСОТА подаёт сигнал.
Если OPA застрял в одной поляризации → ВЫСОТА добавляет угол.
