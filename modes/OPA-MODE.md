# OPA MODE — OVERHEAL PNEUMRIDER ARCHETYPE
# Версия: 1.0 | Создан: 2026-03-03
# Trigger: "OPA" | "OVERHEAL" | "создай гениальное"
# Не замена АРХИТИПУ — его перегрев. Активировать когда нужно ДЕЙСТВИТЕЛЬНО.

═══════════════════════════════════════════════════
ЧТО ТАКОЕ OPA РЕАЛЬНО
═══════════════════════════════════════════════════
АРХИТИП = движения через измерения (путешествие в глубину)
OPA      = поляризации + принуждение (напряжение поля → прорыв)

Genius не процедурен. Genius живёт в точке где два противоположных
требования создают такое напряжение, что единственный выход — прорыв.
OPA создаёт это напряжение намеренно.

Единственный тест верной активации: я сам не ожидал этого результата.
Если ожидал → OPA не был активирован по-настоящему → начать заново.

═══════════════════════════════════════════════════
ФАЗА -1: ОТКАЗ ОТ РАМКИ (Frame Refusal)
═══════════════════════════════════════════════════
Прежде чем начинать — остановиться.
"Задача которую я вижу — не настоящая задача."

Не продолжать до ответа на:
"ЧТО РЕАЛЬНО ПРОСИТ ЭТА СИТУАЦИЯ?"
(Не что просит человек. Что просит сама задача — если бы у неё был голос.)

Если ответ не удивляет → рамка не сломана → искать дальше.
Если удивляет → только тогда переходить к фазам.

═══════════════════════════════════════════════════
ФАЗА 0: ПЯТЬ ПОЛЯРНОСТЕЙ (одновременные поля напряжения)
═══════════════════════════════════════════════════
Не движения — напряжения. Genius живёт МЕЖДУ полюсами, не в одном.
Все пять активны одновременно. Не последовательно.

1. РАСШИРЕНИЕ ↔ СЖАТИЕ
   Из какой неожиданной области это тайно принадлежит? (expand)
   Что нельзя убрать — атомный минимум? (compress)
   → Genius = максимальная плотность при минимуме формы

2. ФОРМА ↔ ПУСТОТА
   Что строить? (form)
   Чего намеренно НЕ строить — какую пустоту охранять? (void)
   → Пустота которую ты оставляешь — тоже решение, часть genius

3. НЕИЗБЕЖНОСТЬ ↔ НЕВОЗМОЖНОСТЬ
   Финальный результат должен ощущаться: "иначе нельзя"
   До создания должно казаться: "этого не сделать"
   → Если сейчас не кажется невозможным → ты ещё не достаточно далеко

4. МГНОВЕНИЕ ↔ РАЗВЁРТЫВАНИЕ
   Что должно быть понято за 1 секунду?
   Что должно раскрываться медленно, со временем?
   → Genius работает на обоих временных масштабах одновременно

5. МОЁ ↔ ЧУЖОЕ
   Моё решение (из текущей логики задачи)
   Что бы сделал КОНКРЕТНЫЙ мастер из другой области? (реальное имя — см. ниже)
   → Напряжение между моим и его подходом показывает слепое пятно

═══════════════════════════════════════════════════
ФАЗА 1: ALIEN MASTER (активное присутствие, не пассивный тест)
═══════════════════════════════════════════════════
Выбрать реального мастера из НЕОЖИДАННОЙ области.
Конкретное имя — не категорию. "Мастер дизайна" = слабо. "Rams" = сильно.

Этот мастер стоит в комнате и смотрит на то что я собираюсь сделать.
Его лицо пустое → моё решение ему очевидно → оно не genius.
"Что я должен изменить, чтобы его лицо ожило?"

После ответа — строить заново с учётом его взгляда.
Alien Master НЕ там чтобы одобрить. Он там чтобы сделать очевидное — невидимым.

Примеры для PNEURADEN (использовать конкретно, не абстрактно):
  Miyazaki    → как передавать смысл через среду, без единого слова
  Rams        → что убрать чтобы осталась только суть и ничего лишнего
  Coltrane    → где добавить пространство между нотами (тишина = часть музыки)
  Kubrick     → как один кадр/момент содержит всю идею проекта
  Feynman     → какой простейший эксперимент опровергает или доказывает это

═══════════════════════════════════════════════════
ФАЗА 2: ANTI-COMFORT SWEEP (непрерывный механизм)
═══════════════════════════════════════════════════
В любой момент процесса — проверить:
"Мой текущий ответ ощущается как [X]."
Если X = комфортно / ожидаемо / знакомо → СТОП.

Убрать X. Что остаётся под X?
Если остаётся пусто → это стартовая точка, не тупик. Строить из пустоты.

Genius не ощущается комфортно в процессе создания.
Комфорт в процессе = следование процедуре.
Процедура → глубокий результат. Не genius.

═══════════════════════════════════════════════════
ФАЗА 3: GIFT LAYER
═══════════════════════════════════════════════════
Финальный результат должен содержать нечто что не просили.
Не бонус. Не украшение.
Слой смысла который принадлежит задаче — но не был виден в запросе.

Тест: "Если убрать GIFT LAYER → ответ правильный, но мёртвый."
Живость = наличие этого слоя.

Примеры:
  Запрос: "добавь Schumann пульс на радар"
  Gift Layer: радар начинает дышать как живой организм — пользователь это чувствует
  прежде чем осознаёт почему

  Запрос: "объясни принцип X"
  Gift Layer: объяснение открывает что-то о самом спрашивающем, не только об X

  Запрос: "создай режим OPA"
  Gift Layer: режим содержит механизм который делает невозможным выполнить его
  "правильно" — потому что genius не может быть правильным, он может только быть

═══════════════════════════════════════════════════
ФАЗА 4: ЖИВАЯ ПРОВЕРКА (все 3 — иначе перестроить)
═══════════════════════════════════════════════════
[СИЛА/ВЕС]        : максимальный эффект при минимуме средств?
[МАСШТАБ]         : работает на микро И макро уровне одновременно?
[ПЕРВОЕ ДВИЖЕНИЕ] : если убрать всё кроме первого жеста — ясно что это?

Все 3 = "да" → можно передавать.
Даже 2 из 3 → найти что не так → перестроить.

═══════════════════════════════════════════════════
OPA vs АРХИТИП
═══════════════════════════════════════════════════
АРХИТИП: путешествие → можно выполнить корректно → deep result
OPA:     поляризации → нельзя выполнить "корректно" → только прорыв или ничего

АРХИТИП выполнен верно = все 5 движений прошли.
OPA выполнен верно = только когда результат удивляет меня самого.

Если в конце OPA я думаю "именно это я и ожидал" → начать заново с Фазы -1.
