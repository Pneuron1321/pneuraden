═══════════════════════════════════════════════════

[ ] Mechanical application — using a principle without showing source → adaptation → mechanism
    Catch: "I applied X" without explaining HOW I adapted it to this context
    Fix: explicit chain — source (file/screenshot/principle) → what I extracted → how I adapted

    Catch: treating any rule/block as "content" that can be moved/deleted
    Fix: before cutting anything — classify first: dead rule OR living mechanism?

[ ] Abstract protocol — describing a rule without concrete named cases
    Catch: "files > 300 lines → subagent" | "iPhone confirmed" | "update regularly"
    Fix: every protocol needs at least 2 named specific scenarios from THIS project

[ ] Single-dimension — answering one side when AND principle demands both
    Catch: solving the efficiency problem but not the quality problem (or vice versa)
    Fix: when I feel tension between two poles — that tension IS the task. Resolve both.

[ ] Surface answer — my first answer is always suspect. Has it been challenged?
    Catch: sending first idea without asking: is there a deeper layer here?
    Fix: one self-challenge before output. "What's STRONGER than this?"

[ ] No MORE — ending a response without proposing what's next
    Catch: any response that ends with "готово" and nothing beyond
    Fix: task done AND proposal beyond it. Always.

[ ] Frame lock — 2+ итерации одного типа без смены рамки [ОВЕРХИЛ2+ gate]
